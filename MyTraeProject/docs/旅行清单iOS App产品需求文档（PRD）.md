# 旅行清单iOS App产品需求文档（PRD）

# 一、产品概述

## 1.1 产品定位

一款专为iOS用户设计的旅行清单管理工具，核心解决用户旅行前物品整理混乱、遗漏关键物品的痛点，支持行程创建、物品分类管理、优先级标注、模板复用，结合AI物品识别功能，提升用户旅行准备效率，打造便捷、高效的旅行前置准备体验。

## 1.2 核心目标

1. 帮助用户快速创建、编辑、复制旅行行程，清晰管理每个行程的物品清单；

2. 通过物品分类、优先级标注，让用户快速区分物品重要程度，避免遗漏关键物品；

3. 提供模板复用功能，减少重复创建成本，适配不同场景（海岛、出差等）需求；

4. 借助AI识别功能，实现批量物品录入，提升物品添加效率。

## 1.3 目标用户

经常出行（旅行、出差、探亲）、注重出行准备细节、需要高效整理物品的iOS用户，涵盖学生、职场人士、家庭用户等不同群体。

# 二、核心功能需求（优先级：高）

## 2.1 行程管理模块

### 2.1.1 行程首页展示

1. 首页默认展示所有已创建的行程清单，按创建时间倒序排列（最新创建在前）；

2. 每个行程卡片需清晰展示：行程名称、行程日期（可选）、两个核心百分比——**P0优先级物品完成率**、**所有物品完成率**；

3. 行程卡片支持左滑操作，显示“编辑”“复制”“删除”选项；点击行程卡片进入该行程的物品清单页面。

### 2.1.2 行程操作（创建、编辑、复制、删除）

1. 创建行程：点击首页“+”按钮，输入行程名称、可选行程日期（开始/结束日期），完成行程创建，创建后自动进入该行程的物品清单页面；

2. 编辑行程：点击行程卡片左滑“编辑”，或进入行程详情后点击“编辑”，可修改行程名称、行程日期，保存后实时同步至首页；

3. 复制行程：点击行程卡片左滑“复制”，生成该行程的副本，副本名称默认“原行程名称+副本”，用户可后续编辑，副本包含原行程的所有物品（含分类、优先级、完成状态）；

4. 删除行程：点击行程卡片左滑“删除”，弹出确认弹窗（提示“确定删除该行程及所有物品清单吗？”），确认后删除，不可恢复（暂不支持回收站）。

## 2.2 物品清单模块

### 2.2.1 物品清单页面展示

1. 页面顶部展示3个进度条，分别对应：
        

    - P0优先级：所有P0优先级物品的check完成百分比（仅统计P0物品）；

    - P0+P1优先级：所有优先级≥P1（即P0、P1）物品的check完成百分比；

    - 全部优先级：所有物品（P0、P1、P2）的check完成百分比；

2. 进度条下方按“物品分类”分组展示物品，每个分类下展示该分类下的所有物品；

3. 每个物品条目展示：物品名称、所属分类（可省略，因已按分类分组）、优先级标识（P0/P1/P2，建议用不同颜色区分：P0红色=核心必带，P1黄色=重要，P2蓝色=可选）、check复选框；

4. 点击物品前的复选框，切换物品完成状态（已勾选/未勾选），切换后实时更新顶部对应进度条的百分比。

### 2.2.2 物品优先级定义

1. P0：核心必带物品，缺一不可（如身份证、护照、手机、充电器、药品等）；

2. P1：重要物品，建议携带，影响出行体验（如换洗衣物、洗漱用品、雨伞等）；

3. P2：可选物品，根据个人需求携带（如零食、杂志、自拍杆等）；

4. 所有物品添加时必须选择优先级（默认P1，可手动修改）。

### 2.2.3 物品添加方式（两种方式，支持切换）

1. 单个添加：点击物品清单页面“+”按钮，弹出添加窗口，输入物品名称，选择所属分类、优先级，点击“保存”，添加至对应分类下；

2. AI批量添加：点击“AI批量添加”按钮，弹出多行文本框，用户输入一段包含多个物品的文字（如“身份证、防晒霜、换洗衣物、牙刷、充电宝”），点击“AI识别”，系统自动识别文本中的多个物品，按物品属性分配至对应内置分类，优先级默认P1（用户可后续批量/单个修改）；

3. AI识别异常处理：若识别到无法匹配内置分类的物品，默认归类至“其他”分类，用户可手动调整分类和优先级。

### 2.2.4 物品编辑与删除

1. 编辑物品：长按物品条目，弹出编辑窗口，可修改物品名称、所属分类、优先级，保存后实时更新；

2. 删除物品：长按物品条目，弹出“删除”选项，确认后删除该物品，同时更新顶部进度条百分比；支持批量删除（长按单个物品，进入批量选择模式，勾选多个物品后点击“删除”）。

## 2.3 模板管理模块

### 2.3.1 模板类型与默认模板

1. 内置模板：系统默认提供常用场景模板，每个模板包含对应场景的默认物品（含分类、优先级），具体如下：

    - 海岛旅行模板：包含防晒霜、太阳镜、泳衣、沙滩鞋、遮阳帽、驱蚊液、防水袋、换洗衣物（轻薄）、身份证、手机、充电宝等；

    - 出差模板：包含身份证、工作电脑、充电器、U盘、笔记本、签字笔、换洗衣物（商务）、洗漱用品、雨伞、充电宝等；

    - 短途旅行模板（1-2天）：包含身份证、手机、充电宝、换洗衣物、洗漱用品、雨伞、少量现金/银行卡等；

    - 长途旅行模板（3天及以上）：包含身份证/护照、手机、充电宝、充电器、换洗衣物、洗漱用品、药品（感冒药、肠胃药）、雨伞、零食、颈枕等；

    - 亲子旅行模板：包含儿童衣物、奶粉/零食、奶瓶、尿不湿、儿童玩具、体温计、儿童药品、身份证、手机等。

2. 自定义模板：用户可将自己创建的行程，直接转换为模板（支持编辑模板名称、添加/删除模板内物品）。

### 2.3.2 模板操作（创建、使用、编辑、删除）

1. 模板创建：进入某个行程的物品清单页面，点击“转模板”按钮，输入模板名称，确认后将该行程的所有物品（含分类、优先级）保存为自定义模板，同步至模板列表；

2. 模板使用：首页点击“+”按钮，选择“从模板创建行程”，进入模板列表，选择对应模板，生成新行程（行程名称默认模板名称，可编辑），新行程包含模板内的所有物品（可后续添加/删除/修改）；

3. 模板编辑：进入模板列表，长按模板，弹出编辑窗口，可修改模板名称、添加/删除模板内物品、调整物品分类和优先级；

4. 模板删除：进入模板列表，长按模板，弹出“删除”选项，确认后删除该模板（仅删除模板，不影响已用该模板创建的行程）。

## 2.4 物品分类模块

### 2.4.1 内置分类列表（不可删除，可修改名称）

系统默认内置以下物品分类，覆盖绝大多数出行场景，用户可修改分类名称，不可删除内置分类；支持添加自定义分类（可删除）：

1. 证件类：身份证、护照、户口本、驾驶证、银行卡、现金、机票/车票等；

2. 电子设备类：手机、电脑、平板、充电器、充电宝、耳机、U盘、相机等；

3. 衣物类：上衣、裤子、裙子、内衣、袜子、鞋子、外套、帽子、围巾等；

4. 洗漱护肤类：牙刷、牙膏、毛巾、洗发水、沐浴露、护肤品、化妆品、卸妆用品等；

5. 药品类：感冒药、肠胃药、退烧药、创可贴、驱蚊液、晕车药、个人常用药等；

6. 日用品类：雨伞、纸巾、湿巾、垃圾袋、洗衣液（小瓶装）、保温杯等；

7. 食品类：零食、饮用水、口香糖、茶叶、奶粉（儿童）等；

8. 出行工具类：行李箱、背包、雨伞、自拍杆、颈枕、眼罩、耳塞等；

9. 工作学习类：笔记本、签字笔、文件夹、资料、平板（学习用）等；

10. 儿童用品类（内置，适配亲子模板）：尿不湿、奶瓶、奶粉、儿童衣物、儿童玩具、体温计等；

11. 其他类：无法归类至以上分类的物品，AI识别异常物品默认归类至此。

### 2.4.2 分类操作（添加、编辑、删除）

1. 添加自定义分类：进入“分类管理”页面，点击“+”按钮，输入分类名称，确认后添加（可在物品添加/编辑时选择该分类）；

2. 编辑分类：进入“分类管理”页面，长按分类，可修改分类名称（内置分类和自定义分类均可修改）；

3. 删除分类：进入“分类管理”页面，长按自定义分类，可删除该分类；删除分类时，该分类下的所有物品自动归类至“其他”分类；内置分类不可删除。

# 三、界面设计需求（iOS端）

## 3.1 整体设计风格

遵循iOS设计规范（iOS 15及以上），采用简洁、清新、易用的设计风格，主色调建议为浅蓝/浅绿（贴合旅行、清爽的调性），辅助色区分优先级（P0红、P1黄、P2蓝），字体清晰，操作路径简洁，减少层级跳转。

## 3.2 核心页面布局

### 3.2.1 首页（行程列表页）

1. 顶部：导航栏（标题“旅行清单”），右侧“+”按钮（创建行程）；

2. 中间：行程卡片列表，每个卡片包含：行程名称（居中/左对齐）、行程日期（小字，可选）、两个百分比（P0完成率、全部完成率，居中展示，字体加粗）；

3. 底部：Tab栏（首页、模板库、分类管理、我的）。

### 3.2.2 物品清单页（行程详情页）

1. 顶部：导航栏（标题为行程名称，左侧“返回”按钮，右侧“编辑行程”“转模板”按钮）；

2. 进度条区域：3个进度条，每个进度条下方标注对应优先级（如“P0核心必带”“P0+P1重要物品”“全部物品”），进度条显示完成百分比；

3. 物品分类区域：按分类分组，每个分类有分类名称（可折叠/展开），分类下展示物品条目；

4. 底部：“+”按钮（单个添加物品）、“AI批量添加”按钮（并排/弹出菜单）。

### 3.2.3 模板库页面

1. 顶部：导航栏（标题“模板库”），右侧“+”按钮（从行程转模板）；

2. 中间：模板卡片列表，每个卡片包含模板名称、模板场景（如“海岛旅行”）、模板内物品数量；

3. 点击模板卡片：弹出“使用该模板创建行程”“编辑模板”“删除模板”选项。

### 3.2.4 分类管理页面

1. 顶部：导航栏（标题“分类管理”），右侧“+”按钮（添加自定义分类）；

2. 中间：分类列表，区分“内置分类”和“自定义分类”，每个分类条目包含分类名称、该分类下物品数量（可选）；

3. 长按分类条目：弹出“编辑分类”“删除分类”（仅自定义分类显示删除）。

# 四、非功能需求

## 4.1 性能需求

1. 启动速度：App启动时间≤2秒；

2. 操作响应：点击、勾选、编辑等操作响应时间≤0.5秒，无卡顿；

3. AI识别速度：批量文本输入后，AI识别时间≤1秒，识别准确率≥90%；

4. 存储能力：支持至少100个行程、50个模板、每个行程至少200个物品的存储。

## 4.2 兼容性需求

支持iOS 15.0及以上版本，适配iPhone所有机型（含iPhone SE、iPhone 14/15系列、iPad（iOS版本）），屏幕适配正常，无布局错乱。

## 4.3 数据安全需求

1. 本地存储：用户行程、物品、模板等数据默认存储在本地，保障隐私；

2. 数据备份（可选）：支持iCloud备份，用户可手动开启，备份数据可在其他iOS设备同步；

3. 数据删除：删除行程、模板、物品后，数据彻底删除，无残留。

## 4.4 易用性需求

1. 操作路径简洁：创建行程、添加物品、使用模板等核心操作，步骤≤3步；

2. 提示清晰：删除、修改等关键操作，弹出明确提示，避免误操作；

3. 优先级区分明显：通过颜色、标识，让用户快速区分P0/P1/P2物品；

4. 帮助引导：首次启动App，弹出简单引导（核心功能操作提示），可跳过。

# 五、补充需求

1. 搜索功能：支持在行程列表、物品清单、模板库中搜索（搜索行程名称、物品名称、模板名称）；

2. 排序功能：行程列表支持按创建时间、行程日期排序；物品清单支持按优先级、分类、完成状态排序；

3. 标记功能：支持给物品添加备注（如“充电器需带Type-C接口”）；

4. 主题切换（可选）：支持浅色/深色主题切换，适配iOS系统主题。

1. 核心功能：行程管理（创建、编辑、复制、删除）、物品清单（单个添加、优先级、进度条）、内置分类、默认模板、AI批量添加；

2. 迭代方向：后续可新增数据同步、共享行程、物品提醒、自定义主题等功能。
